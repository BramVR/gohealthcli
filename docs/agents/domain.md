# Domain Docs

This is a single-context repo.

Before producing implementation issues or architecture guidance, read:

- `CONTEXT.md` for glossary language.
- `docs/adr/` for accepted decisions relevant to the work.
- `docs/plan.md`, `docs/data-model.md`, `docs/provider-google-health.md`,
  and `docs/security.md` when their `read_when` hints match the task.
- `docs/commands.md` (no `read_when` frontmatter — it is regenerated by
  `make docs-commands`) when the CLI surface matters to the task.

Use glossary vocabulary in issue titles and descriptions. If a proposed issue
contradicts an ADR, call that out explicitly instead of silently overriding it.
